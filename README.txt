وكالة الدواحي للسفر والسياحة
================================

⚠️ سبب مشكلة "السيرفر" على GitHub
-----------------------------------
GitHub نفسه (وخصوصاً GitHub Pages) بيستضيف ملفات ثابتة فقط (HTML/CSS/JS).
هو مش قادر يشغّل كود بايثون (Flask) ولا قاعدة بيانات SQLite.
فلو رفعت المشروع على GitHub وفتحته عن طريق GitHub Pages، هتلاقي الصفحات
بتظهر لكن أي زرار (تسجيل دخول / حجز / إنشاء حساب) هيفشل، لأن مفيش
سيرفر شغال يستقبل طلبات /api/... دي.

الحل: عندك خياران
--------------------
1) التشغيل محلياً (زي ما كان مخطط له أصلاً عبر Termux):
   - GitHub هنا بيبقى بس مكان لتخزين الكود (git clone / git pull)
   - بعد ما تنزل الكود على الهاتف أو الكمبيوتر، تشغّله بـ:
       pip install -r requirements.txt
       python app.py
   - هيفتح على http://127.0.0.1:5000

2) الاستضافة الحقيقية على الإنترنت (لو عايز الموقع يشتغل بدون ما جهازك
   يكون مفتوح)، استخدم استضافة بتدعم بايثون زي Render أو Railway
   (مش GitHub Pages):
   - ارفع المشروع على GitHub عادي (git push)
   - اربط الـ repo بـ Render.com أو Railway.app (فيه خطط مجانية)
   - هما هيقروا ملف requirements.txt و Procfile تلقائياً ويشغلوا:
       gunicorn app:app
   - غيّر متغير app.secret_key في app.py لقيمة سرّية عشوائية قبل النشر
     (السطر فيه تعليق يوضح ده).

ملاحظات مهمة عن الأمان قبل الرفع على GitHub
-----------------------------------------------
- ملف dawahi.db (قاعدة البيانات) وصور الجوازات في uploads/passports
  مستثناة من الرفع تلقائياً عن طريق .gitignore، عشان بيانات المستخدمين
  الحقيقية متتسربش على GitHub.
- لو المشروع كان مرفوع قبل كده على GitHub بأي بيانات حقيقية، لازم
  تتأكد إنها اتمسحت من تاريخ الـ git كمان مش بس من آخر نسخة.

الفحص الذي تم إجراؤه
------------------------
تم تشغيل السيرفر فعلياً واختبار كل نقاط الـ API:
  - إنشاء حساب (/api/signup) ✅
  - تسجيل الدخول (/api/login) ✅
  - الجلسة (/api/session) ✅
  - إنشاء حجز (/api/book) ✅
  - عرض الحجوزات (/api/bookings) ✅
  - تفاصيل حجز (/api/booking/<code>) ✅
  - إلغاء حجز (/api/booking/<code>/cancel) ✅
  - تسجيل الخروج (/api/logout) ✅
  - كل صفحات HTML وملفات CSS/JS بترجع 200 ✅
كل النتائج جت صحيحة بدون أي أخطاء.

خطوات التشغيل الأصلية (Termux) لسه زي ما هي
-----------------------------------------------
1) pkg update && pkg install python -y
2) pip install -r requirements.txt
3) انسخ مجلد المشروع للهاتف وادخل عليه: cd مسار/المجلد
4) python app.py
5) افتح من متصفح الهاتف: http://127.0.0.1:5000
